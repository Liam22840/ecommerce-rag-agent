# 提交文档入口

最终提交建议使用下面三份飞书文档。仓库里的 Markdown 已按飞书可复制的结构整理，复制到飞书后只需要补仓库地址、飞书链接和演示视频链接。

| 飞书文档 | 仓库源文件 | 用途 |
| --- | --- | --- |
| 项目文档 | [`ProjectDocument.md`](ProjectDocument.md) | 按比赛模板填写队伍、分工、项目、代码仓库、设计文档链接、说明文档链接、演示视频和项目亮点。 |
| 设计文档 | [`DesignDocument.md`](DesignDocument.md) | 说明产品目标、系统架构、RAG 方案、智能体编排、事实安全、客户端体验和关键设计取舍。 |
| 说明文档 | [`UsageDocument.md`](UsageDocument.md) | 说明如何部署和体验产品，以及评审如何快速复现核心流程。 |

## 飞书链接填写位置

复制到飞书后，建议在项目文档里填写：

- 设计文档：填写“设计文档”飞书链接。
- 说明文档：填写“说明文档”飞书链接。
- 代码仓库：填写 GitHub 仓库地址或代码压缩包地址。
- 演示视频：填写 5-10 分钟讲解视频链接。
