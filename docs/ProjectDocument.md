## 项目简介

该项目是一个基于检索增强生成的多模态电商导购助手。用户可以用自然语言、语音或图片表达购物需求，系统会在真实商品库中检索商品，并给出推荐、对比、加购和模拟下单结果。

客户端使用 SwiftUI 原生实现，后端使用 FastAPI 提供流式接口，商品检索基于 Milvus Lite 和豆包多模态向量模型。系统的核心原则是：模型负责理解用户意图，代码负责核对事实和执行状态。价格、SKU、库存、购物车金额和订单状态都从结构化数据中读取，模型不能自由编造。

## 运行和体验入口

评审可以按说明文档启动后端和 iOS 应用。推荐演示顺序如下：

| 场景 | 用户文案 | 预期展示 |
| --- | --- | --- |
| 基础推荐 | 推荐一款适合油皮的洗面奶 | 流式回答和商品卡片。 |
| 条件筛选 | 200 元以下的蓝牙耳机有哪些？ | 预算内耳机结果。 |
| 主动澄清 | 推荐一款手机 | 系统先询问预算、拍照、续航等方向。 |
| 多轮收敛 | 拍照优先，预算 4000 | 继承手机主题继续推荐。 |
| 对比决策 | 第一个和第二个哪个更便宜？ | 结构化对比和具体 SKU 价格。 |
| 购物车 | 把更便宜的加入购物车 | 更新购物车状态。 |
| 商品详情和收藏 | 打开商品详情并收藏第一款 | 查看详情分页、收藏数量和收藏页。 |
| 下单 | 下单，然后把地址改成上海徐汇区漕溪北路 18 号 | 生成订单草稿并更新地址。 |
| 多步任务 | 帮我推荐跑鞋，对比最便宜的两双，然后把更适合日常跑步的加入购物车 | 先显示计划，再实时更新步骤。 |

| 语音体验 | 用麦克风说“推荐一款适合干皮的面霜” | 语音识别后走正常推荐，并支持播报。 |

## 项目亮点

1. **事实安全的电商 RAG**  
   系统把模型能力限制在“理解和表达”，价格、SKU、库存、购物车和订单都由代码核验，减少电商场景最严重的事实错误。

2. **可执行的多智能体购物闭环**  
   系统不只回答推荐问题，还能执行对比、加购、改数量、改地址、下单和确认。Planner 支持一句话完成多步任务，并在客户端实时展示步骤。

3. **原生多模态购物体验**  
   iOS 端实现 SwiftUI 流式对话、商品卡片、详情页、收藏、拍照找货、语音输入、TTS 播报、购物车和订单确认，并补齐骨架屏、滑动操作、加购飞行动效和订单成功反馈，体验接近真实导购产品。

## 其他补充信息

- 库存是演示用合成库存，不连接真实仓储。
- 下单是模拟订单，不连接真实支付和物流。
- 商品库规模是一百余个商品，主要验证工程链路和 RAG 可靠性。
- TTS 云端播报依赖 Gemini TTS 密钥；没有密钥时 iOS 使用系统语音兜底。
